Touch

Duration

10 minutes

Tip: If you are doing this exercise live in a session, make sure to make good use of the instructor, they are online to answer any questions you have!

Goals

In this lab, you will run a pre-built app to see when each of the 5 MotionEventActions enum values are reported for multi-touch. There is no code to write in this exercise.

Required assets

You will need the solution contained in the Completed Resources folder for this part.

Steps

Below are the step-by-step instructions for the exercise.

  1. Locate the Resources folder for this part. Open the MultiTouchActionsTest solution in the Completed folder.
  2. Run the app. A physical device is best for this; however, any emulator that lets you simulate more than one touch point will work (e.g. the Xamarin Android Player).
  3. The UI will be empty when you first launch the app. As you touch the screen, the app will add entries to a ListView. The list entries show you the value of ActionMasked and the current number of touch points as each event occurs.
  4. Touch and release with a single finger. You should see two list entries showing you a Down action followed by an Up action. If you dragged your finger across the surface, you may also see a few Move actions. In all cases, you should see that only 1 pointer is currently in contact with the screen. The Down and Up actions report first-finger-down and last-finger-up so they are exactly what you should see for single-touch.
  5. Click the button in the upper-right corner of the UI (in the Action Bar). This will clear the content of the ListView.
  6. Touch with a single finger. Touch with a second finger. Release with one finger. Release with the other finger. You should see the sequence: Down, PointerDown, PointerUp, and Up. The pointer count should match the number of fingers you have touching the screen at each step. If you are using the Xamarin Android Player, hold down the Alt/Option key and then click to simulate two-finger-touch (you may have to drag your finger a little bit to get the emulator to report the second touch).
  7. If you are using a physical device, you can experiment with more than two fingers; however, please keep in mind that the ListView will quickly run out of screen space to display the output.

Summary

This exercise let you verify the MotionEventActions that Android reports for multi-touch. Your app receives Down when the first finger touches the screen, PointerDown for each additional finger, PointerUp when non-primary fingers leave the screen, and finally Up when the use removes the last finger.
